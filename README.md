# مجموعة بيانات التفسير المحرر (Tafsir Al-Muharrar Dataset)

مجموعة بيانات (Dataset) مفتوحة ومحققة برمجياً لموسوعة **التفسير المحرر** الصادرة عن **مؤسسة الدرر السنية**. يوفر المشروع المادة العلمية كاملة بصيغ مهيكلة (HTML و JSON) لتسهيل مهام البحث العلمي، ومعالجة اللغات الطبيعية (NLP)، وتطبيقات الذكاء الاصطناعي والتعليم الإسلامي، مع تزويدها بأدوات عرض وتحقق محلية.

> [!IMPORTANT]
> **تنبيه حقوقي**: المادة العلمية ومحتواها أصلها موسوعة التفسير المحرر الصادرة عن مؤسسة الدرر السنية. يتعين على مستخدم البيانات مراعاة شروط استخدام المصدر الأصلي وعزو المادة إلى أهلها عند أي استخدام خارجي.

> [!WARNING]
> **إخلاء مسؤولية (Disclaimer)**:
> أُنشِئت مجموعة البيانات هذه لأغراض البحث العلمي والتجربة البرمجية. وعلى الرغم من إعمال أدوات التحقق التلقائي ومقارنة البصمات، إلا أن مخرجات JSON الحالية لم تخضع لمراجعة وتدقيق بشريين كاملين.
> وننوه بأن سلامة البيانات ودقتها مرهونة بصحة السكربتات البرمجية المسؤولة عن الاستخلاص والتوليد. وبسبب عدم اعتماد مطوري موقع "الدرر السنية" (المصدر) على حقول ومعرفات HTML ثابتة، وحدوث أخطاء واضطرابات متكررة في بنية الصفحات، فقد اضطررنا لمعالجة ذلك برمجياً عبر قواعد استثنائية معقدة داخل السكربتات، وهو ما قد يعرض المخرجات لبعض الأخطاء أو السقوط غير المتوقع.
> وبناءً عليه، لا نضمن خلو هذه المخرجات من الأخطاء، ويُنصح بمراجعة الموقع الرسمي لمؤسسة الدرر السنية أو الأمهات المطبوعة للتيقن عند الاحتجاج العلمي أو الفتوى الشرعية.

## أصول البيانات المحلية (`tafseer html/`)

تمثل مجلدات `tafseer html/` المستودع الخام للبيانات المستخلصة؛ حيث تُفرد لكل سورة حافظة مستقلة تحتوي على:
* `المقدمة.html`: يحوي ديباجة السورة وفضائلها ومقاصدها.
* ملفات النطاقات (مثل `1-5.html` أو `12.html`): تحوي صفحات التفسير الخاصة بالآيات المعنية.

> [!NOTE]
> هذا المجلد متجاهل محلياً في مستودع Git لكبر حجمه. يمكن للراغبين في بناء البيانات محلياً تحميلها وتوليدها باستخدام الأدوات المرفقة كما هو موضح في دليل التوليد أدناه.

## طريقة البدء وتوليد البيانات محلياً

إذا كنت ترغب في جلب البيانات الخام وتوليد ملفات JSON بنفسك، يمكنك اتباع الخطوات التالية:

1. **تحميل صفحات HTML الخام**:
   قم بتشغيل سكربت التحميل للبدء في جلب الصفحات من موقع الدرر السنية وحفظها في مجلد `tafseer html/`:
   ```powershell
   python scripts/downloader.py
   ```
   *(يمكن إيقاف السكربت واستئنافه في أي وقت؛ حيث يتتبع التقدم تلقائياً).*

2. **توليد ملفات JSON المهيكلة**:
   بعد اكتمال التحميل، قم بتشغيل سكربت التوليد لاستخلاص النصوص وبناء ملفات JSON الفردية والملف الموحد:
   ```powershell
   python scripts/json_generator.py --all --no-progress
   ```

3. **التحقق من سلامة البيانات**:
   لمطابقة البيانات المتولدة مع الأصول والتأكد من عدم وجود أي نقص أو خلل بنيوي:
   ```powershell
   python scripts/validator.py --unified
   ```

## مواصفات مخرجات JSON (`json/`)

يضم مجلد `json/` ملفات السور المستقلة (`1.json` إلى `114.json`). وتخضع البيانات لقواعد صارمة تمنع تسرب وسوم HTML وتفصل المتون عن الحواشي:
* **تجريد النصوص**: تُحفظ الحقول كنصوص ظاهرة خالية من وسوم التنسيق البرمجية.
* **إحالات الحواشي**: يُبقى في المتن رقم الحاشية المختصر كرمز ربط (مثال: `[15]`).
* **عزل المراجع**: تنقل النصوص الكاملة للحواشي إلى قائمة `references` الموزعة حسب الأقسام (التفسير، البلاغة، الإعراب، غريب الكلمات... إلخ).

لمزيد من التفاصيل، يرجى مراجعة [مواصفات_JSON.md](file:///e:/GitHub/my%20workflows/%D8%A7%D9%84%D8%AA%D9%81%D8%B3%D9%8A%D8%B1%20%D8%A7%D9%84%D9%85%D8%AD%D8%B1%D8%B1/%D9%85%D9%88%D8%A7%D8%B5%D9%81%D8%A7%D8%AA_JSON.md).

## الضوابط الهندسية ومعالجة البيانات (Technical Engineering Rules)

خضعت عملية استخلاص البيانات وتصفيتها لقواعد تقنية صارمة لضمان سلامة المتون ونقاء النصوص:

### 1. دقة تحديد نطاق الآيات
لضمان عدم تعطل الاستخلاص أو تسمية الملفات، يُعتمد تعبير نمطي (Regex) مرن يدعم كافة صياغات النطاقات في صفحات المصدر:
```regex
(?:الآيات|الآية|الآيتان|الآيتين)\s*[:]?\s*[\(]?\s*(\d+)\s*(?:[-–—]\s*(\d+))?\s*[\)]?
```

### 2. معالجة خطوط الآيات العثمانية
تستخدم صفحات المصدر حاوية `div#qpage` لعرض الآيات بخطوط عثمانية خاصة تظهر كرموز مبهمة عند الاستخلاص العادي. لمعالجة ذلك، يقوم السكربت بالبحث عن النص السليم المشكول للآيات من داخل أول وسم `span.aaya` في قسم التفسير ويستبدله بمحتوى حاوية الآيات كاملاً بصيغة اقتباس منسقة.

### 3. تنظيف وتجريد الـ HTML
لضمان خلو ملفات الـ JSON من أي وسوم أو أكواد زائدة، يتم القيام بما يلي:
* **حذف الأكواد البرمجية**: تفكيك وسوم `<style>` و `<script>` بالكامل من شجرة الـ HTML.
* **إزالة عناصر الواجهة**: حذف روابط التبويبات الداخلية والتعليقات المكتوبة بصيغة كلاسات التنقل الجانبية (`nav`, `collapse`, `menu`, `tabs`).
* **تصفية التعليقات والتكرار**: إزالة تعليقات المطورين البرمجية (`Comment` objects) ومعالجة الفراغات والأسطر الفارغة الزائدة لضمان تطابق بصمات SHA-256 بشكل سليم.

## الأدوات البرمجية للحقيبة (Scripts & Tools)

تتوفر في مجلد `scripts/` حزمة من الأدوات البرمجية والسكربتات المساعدة لإدارة وتجهيز مجموعة البيانات:

### 1. أداة جلب وعرض التفسير (`scripts/get_tafsir.py`)
أداة سريعة ومرنة مبنية بواجهة سطر أوامر تفاعلية لعرض التفسير، وغريب الكلمات، والإعراب، والفوائد لسورة معينة أو نطاق محدد من الآيات.

* **الاستخدام الأساسي (عرض السورة كاملة بجميع حقولها وحواشيها)**:
  ```powershell
  python scripts/get_tafsir.py 112
  ```
* **عرض نطاق محدد من الآيات (الآيات من 2 إلى 3)**:
  ```powershell
  python scripts/get_tafsir.py 112 -f 2 -t 3
  ```
* **عرض أقسام معينة فقط دون حواشي (مثال: عرض الآيات والتفسير التفصيلي نظيفاً)**:
  ```powershell
  python scripts/get_tafsir.py 112 --verses --detail --no-ref
  ```
* **خيارات الأقسام المتاحة**:
  * `-i`, `--intro`: عرض المقدمة.
  * `-v`, `--verses`: عرض نص الآيات.
  * `-c`, `--vocab`: عرض غريب الكلمات.
  * `-m`, `--meaning`: عرض المعنى الإجمالي.
  * `-d`, `--detail`: عرض التفسير التفصيلي.
  * `-g`, `--grammar`: عرض الإعراب.
  * `-b`, `--balagha`: عرض البلاغة.
  * `-e`, `--edu`: عرض الفوائد التربوية.
  * `-s`, `--sci`: عرض الفوائد العلمية واللطائف.
  * `--no-ref`: إيقاف عرض الحواشي تماماً.

### 2. سكربت التحميل التلقائي (`scripts/downloader.py`)
يتولى الاتصال برابط التفسير الرئيسي في موقع الدرر السنية وتتبع صفحات كل سورة بالتنقل الآلي وحفظها بصيغة HTML خام داخل مجلد `tafseer html/`، مع تتبع تقدم التنزيل في `progress.json` لتسهيل استئناف العمل عند الانقطاع.

### 3. سكربت توليد الـ JSON (`scripts/json_generator.py`)
يقوم باستخلاص وتوليد ملفات JSON المهيكلة للسور الفردية والملف الموحد `tafseer_all.json` من أصول HTML المحلية الخام في مجلد `tafseer html/`، مع تصفية وسوم HTML وفصل المتون عن المراجع.

* **أمر التشغيل (توليد كافة السور)**:
  ```powershell
  python scripts/json_generator.py --all --no-progress
  ```

### 4. سكربت التحقق والمطابقة (`scripts/validator.py`)
أداة تدقيق بنيوية ومطابقة رقمية تقوم بمقارنة بصمات SHA-256 للمتون في JSON مع أصول HTML في مجلد `tafseer html/`، وفحص مطابقة الحواشي والمراجع وتوزيعها، وضمان سلامة المخطط.

* **أمر التشغيل (التحقق الشامل)**:
  ```powershell
  python scripts/validator.py --unified
  ```

### 5. سكربت الاختبارات المؤتمتة (`scripts/run_tests.py`)
سكربت اختبار شامل يختبر 10 سيناريوهات تشغيل مختلفة لأداة الجلب `get_tafsir.py` للتحقق من سلامة الأوامر وأكواد الخروج ومعالجة الأخطاء.

* **أمر التشغيل**:
  ```powershell
  python scripts/run_tests.py
  ```

### 6. مكتبة الدوال المشتركة (`scripts/helpers.py`)
ملف تجميعي برمجي يحتوي على خوارزميات تنظيف النصوص العربية، وتصفية وسوم HTML، وتصنيف مقالات التفسير والفوائد وتفكيك الحواشي والروابط.
